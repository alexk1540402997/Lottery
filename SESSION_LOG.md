## 求解模式（SolveEngine）

### 5层运算流

```
循环 {
  1. 智能生成一组(参数, 权重)
  2. 对最新N期逐一回测 → 每期命中数
  3. _check_solution: 每期主球≥阈值 AND 辅助球≥阈值？
     ├─ 是 → 收集到 solutions 列表
     └─ 否 → 跳过
  4. 用 closeness（非 avg_hits）更新参数表现追踪
  5. closeness 反馈回采样器
}
```

### 核心判定 `_check_solution()`

```
每期: main_ok = 主球命中 >= tolerance_main
      aux_ok  = 辅助球命中 >= tolerance_aux
      任一期不满足 → 不是解

closeness:
  main_closeness = min(1.0, 主球命中 / 主球容差)
  aux_closeness  = min(1.0, 辅助球命中 / 辅助球容差)
  total_closeness += main_closeness × 0.7 + aux_closeness × 0.3
```

### 为什么 closeness 而非 avg_hits

- 组合 A：平均 6.0 命中，第 3 期只中 2 个 → 不是解
- 组合 B：平均 4.2 命中，每期都中 4 个 → 是解
- avg_hits 会误导搜索向 A 收敛，closeness 惩罚"偏科"

### 局限性

本质仍是蒙特卡罗随机采样+评估，没有真正的求解过程。

#### 权重：离散还是线性？（关键决策）

**合并层的投票运算是线性的：**

```
号码 i 的得票 = Σⱼ w_method[j] × w_gran[k] × I[号码i被方法j在颗粒度k预测]
→ v = M × w  （线性系统）
```

| 场景 | 权重处理方式 | 理由 |
|------|-------------|------|
| **回测模式** | 保持离散（不动） | 找 1 个最优组合，现有方式足够 |
| **求解模式** | 分离：BO 找参数 → 线性求解权重 | 遍历大量组合，线性求解效率碾压 |

**用随机扰动找最优权重 = 用骰子解 Ax=b，不划算。**

### 回测 vs 求解 优化架构

```
回测模式（目标：找1个最优组合）:
  BO建议 → params + weights（一体）
    → evaluate_combo(params, weights)
    → 反馈 avg_hits → 更新 BO

求解模式（目标：找所有满足容差的解）:
  BO建议 → params only（不含权重）
    → evaluate_params_only(params) → 各期各组合的预测
    → 线性权重求解器（NNLS）→ optimal_weights
    → merge_and_score(predictions, optimal_weights)
    → _check_solution() → 收集解
    → 反馈 closeness → 更新 BO
```

## 三大梯队方法全景

### 第一梯队：黑箱优化（直接适用，无需改管线）

**1. 贝叶斯优化（Bayesian Optimization）⭐ 最推荐**

```
原理：GP代理模型 → EI采集函数 → 选最有价值的点评估 → 更新GP
优势：评估次数极少(30-50次)，天然适合昂贵黑箱
依赖：sklearn.gaussian_process.GaussianProcessRegressor（已有）
离散处理：连续松弛 + 四舍五入到最近合法值
```

**2. CMA-ES（协方差矩阵自适应进化策略）**

```
原理：维护 N(μ, σ²C) 分布，每代采样λ个点评估，用最优μ个更新μ和C
优势：自动学习参数相关性，天然批量并行
离散处理：同BO的连续化+舍入
复杂度：~120行NumPy
```

**3. 模拟退火（Simulated Annealing）**

```
原理：随机扰动 → 更好则接受，更差以exp(-ΔE/T)概率接受，T递减
优势：极简(~60行)，天然处理离散，多链可并行
劣势：信息利用率低
```

### 第二梯队：需要改造管线

**4. 响应面方法** — 拟合低阶多项式 → 解析求导 → 验证迭代
**5. 可微替代模型** — Gumbel-Softmax + NeuralSort + 可微决策树 + PyTorch梯度下降
**6. 线性规划/整数规划** — 合并层权重精确求解（固定预测结果前提下）

### 第三梯队：思路启发

**7. SVD/PCA降维** — 参数空间去冗余
**8. 伴随状态法** — 可微管线梯度计算
**9. CSP求解器** — 约束满足+回溯剪枝

## 四种优化器优劣势汇总

### 贝叶斯优化

| 优势 | 劣势 |
|------|------|
| 采样效率最高（30-50次找到优质解） | GP 矩阵求逆 O(n³)，n>500 时变慢 |
| EI 采集函数无需调参 | 离散参数需连续化+舍入 |
| sklearn 自带，无新依赖 | 冷启动需 15-20 个随机样本 |
| 天然输出不确定性估计 | 高维（>50维）时 GP 退化 |

### CMA-ES

| 优势 | 劣势 |
|------|------|
| 自动学习参数相关性 | 每代需评估 λ≈12-15 个点才能更新 |
| 对非凸、多峰鲁棒 | 收敛速度比 BO 慢 |
| 天然批量并行 | 离散场景需舍入 |
| 实现相对简单 | 对噪声敏感（我们无噪声，OK） |

### 模拟退火

| 优势 | 劣势 |
|------|------|
| 实现极简（~60行） | 单链不能并行（多链可以） |
| 理论上全局收敛 | 冷却速率敏感 |
| 天然处理离散参数 | 信息利用率低 |
| 零额外依赖 | 效率不如 BO 和 CMA-ES |

### 线性权重求解

| 优势 | 劣势 |
|------|------|
| 精确求解（非迭代近似） | 只能求解权重，不能替代参数搜索 |
| scipy.nnls 成熟稳定 | 多周期聚合可能无精确可行解 |
| 与 BO/CMAES/SA 互补 | 需分离 predict 和 merge 阶段 |
| 计算极快（<0.01秒） | 假设线性投票模型正确（在合并层确实正确） |

## 可微管线评估

### 结论：不建议做，当前和中期阶段无必要性

| 维度 | 评估 |
|------|------|
| 技术可行性 | 可微近似品存在（NeuralSort, Gumbel-Softmax等） |
| 预测质量风险 | 高。近似品降低各方法独立预测质量 |
| 过拟合风险 | 极高。NN 梯度优化易拟合彩票噪声 |
| 投入产出比 | 极低。数月工作，效果大概率不如当前集成方法 |
| 替代方案 | BO + 线性求解已解决优化问题 |

```
13个独立方法 → 多角度统计捕捉 → 集成投票消除偏差
→ BO优化超参数 + 线性求解权重
→ 数学可解释、工程可控
→ 彩票低信噪比场景下，多样性集成 > 端到端黑箱
```
